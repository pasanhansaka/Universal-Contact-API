How the Backend Server Processes Data

1

CORS Approval (Cross-Origin Resource Sharing)

When the browser sends a request from a portfolio site (e.g. localhost:3000 or pasan.github.io) to your API server (localhost:5000), Express inspects headers. The cors() middleware tells the browser: "Yes, I allow requests from this domain."

2

Payload Parsing & Validation

The Express body parser reads the incoming raw bytes and converts them into a clean JavaScript object: req.body. The server validates that name, email, and message are not empty, returning a 400 Bad Request if they are.

3

SMTP Packet Dispatch (Nodemailer)

The server connects to your SMTP host (e.g. Gmail / Mailtrap) using secret environment credentials from .env. It creates an email packet where the replyTo header is set to the visitor's email, so you can reply directly to them.

4

Final Mail Delivery

The SMTP mail server receives the packet from your Node app, authenticates the sender credentials, and dispatches the email to your personal mailbox (defined in RECEIVER_EMAIL).
